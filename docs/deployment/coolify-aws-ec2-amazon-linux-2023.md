# Coolify Deployment Plan: AWS EC2 Amazon Linux 2023

## Target

- Instance type: t3.micro minimum for MVP.
- Recommended instance type: t3.small if budget allows.
- Operating system: Amazon Linux 2023.
- Deployment platform: self-hosted Coolify.
- Runtime packaging: Docker-based deployments managed by Coolify.
- Frontend: Next.js.
- Backend: Express.js.
- Database: PostgreSQL managed by Coolify on the same EC2 instance for MVP.
- Repository: `https://github.com/doni404/portfolio-nextjs`.

Detailed setup checklist: `docs/deployment/coolify-setup-steps.md`.

## Recommendation

Use Coolify as the primary deployment manager if the server is upgraded from t3.nano.

t3.micro can work for a low-traffic MVP, but it has only 1 GB RAM. Coolify's official docs recommend at least 2 CPU cores, 2 GB RAM, and 30 GB storage. Because t3.micro is below the recommended RAM, add swap space and keep the deployment lean. If builds or PostgreSQL become unstable, upgrade to t3.small.

## What Coolify Should Manage

- GitHub repository connection.
- Production app deployments.
- Environment variables.
- PostgreSQL service.
- Reverse proxy and HTTPS certificates.
- Deployment logs.
- Application restarts.
- Optional database backups.

## What GitHub Actions Should Still Do

GitHub Actions should remain responsible for code quality gates:

- Install dependencies.
- Generate Prisma client.
- Run lint.
- Run typecheck.
- Run tests.
- Build the app.

After CI passes, deployment can happen through:

- Coolify GitHub integration with push-to-deploy, or
- GitHub Actions calling a Coolify deploy webhook.

Recommended MVP choice:

- Use Coolify GitHub integration first.
- Add GitHub Actions deployment webhook later if more control is needed.

## EC2 Security Group

Allow inbound:

- SSH `22` from trusted IP only.
- HTTP `80` from internet.
- HTTPS `443` from internet.
- Coolify setup UI `8000` temporarily during initial setup.

After setup:

- Keep `80` and `443` open.
- Restrict or close direct access to `8000` if Coolify is available through a configured HTTPS domain.
- Do not expose PostgreSQL publicly.

## Server Preparation

Update packages:

```bash
sudo dnf update -y
```

Install basic tools:

```bash
sudo dnf install -y curl git jq openssl
```

Add swap space because t3.micro has limited memory:

```bash
sudo fallocate -l 2G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile
echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab
free -h
```

## Coolify Installation

Official quick install:

```bash
curl -fsSL https://cdn.coollabs.io/coolify/install.sh | sudo bash
```

After installation:

- Open the URL shown by the installer, usually `http://SERVER_IP:8000`.
- Create the first admin account immediately.
- Add the EC2 server in Coolify if it is not already registered.
- Configure a domain for Coolify later if desired.

## Coolify Project Setup

Create a Coolify project:

- Project name: `portfolio-nextjs`
- Environment: `production`

Add resources:

- PostgreSQL database.
- Next.js frontend application.
- Express.js backend application.

## Database

Create PostgreSQL in Coolify:

- Name: `portfolio-postgres`
- Database: `portfolio`
- User: `portfolio_app`
- Password: generated by Coolify.

Set the backend `DATABASE_URL` from the Coolify PostgreSQL connection details.

Keep PostgreSQL private inside the Coolify/Docker network. Do not expose it to the public internet.

## Application Layout

The repository is planned as a monorepo:

```text
apps/
  web/
  api/
packages/
  shared/
```

Coolify can deploy the frontend and backend as separate applications from the same repository.

### Frontend App

- Root directory: `apps/web`
- Build command: `npm run build` or project-specific command.
- Start command: `npm run start`
- Port: `3000`

Required environment variables:

```text
NODE_ENV=production
NEXT_PUBLIC_API_BASE_URL=https://api.doniputra.com
```

If using same-domain routing through Coolify/proxy, the API URL can be:

```text
NEXT_PUBLIC_API_BASE_URL=https://doniputra.com/api
```

### Backend App

- Root directory: `apps/api`
- Build command: `npm run build`
- Start command: `npm run start`
- Port: `4000`

Required environment variables:

```text
NODE_ENV=production
PORT=4000
DATABASE_URL=postgresql://portfolio_app:password@portfolio-postgres:5432/portfolio
FRONTEND_URL=https://doniputra.com
CONTACT_NOTIFICATION_EMAIL=doniputrapurbawa@gmail.com
ADMIN_SEED_PASSWORD=replace-with-strong-initial-password
SESSION_SECRET=replace-with-long-random-secret
```

## Prisma Migration Strategy

Recommended deployment command sequence for the backend:

```bash
npm run prisma:generate
npm run prisma:migrate:deploy
npm run build
```

Seed only when needed:

```bash
npm run db:seed
```

Do not automatically run seed on every production deployment unless the script is proven idempotent and safe for manually edited content.

## Domains

Recommended:

- Public site: `doniputra.com`
- API: `api.doniputra.com`
- Coolify dashboard: `coolify.doniputra.com`

Alternative:

- Public site: `doniputra.com`
- API proxied under: `doniputra.com/api`

The subdomain API approach is simpler in Coolify. The same-domain `/api` approach is cleaner for users but may need extra proxy configuration.

## Backups

Enable Coolify database backups for PostgreSQL.

Recommended backup target:

- S3-compatible storage.

Minimum policy:

- Daily backup retained for 7 days.
- Weekly backup retained for 4 weeks.
- Restore test before relying on backups.

## Monitoring

Monitor:

- EC2 CPU credit balance.
- Memory usage.
- Disk usage.
- Docker container restarts.
- PostgreSQL storage.
- Coolify deployment logs.

Upgrade to t3.small if:

- Builds fail due to memory.
- PostgreSQL is killed or restarts.
- Coolify becomes slow or unresponsive.
- The app receives consistent traffic.

## Release Flow

Recommended MVP flow:

1. Push to GitHub.
2. GitHub Actions runs CI.
3. If CI passes, merge to `main`.
4. Coolify deploys from `main`.
5. Coolify runs backend migration command.
6. Run smoke checks:
   - `https://doniputra.com`
   - `https://api.doniputra.com/api/health` or `https://doniputra.com/api/health`
   - `/blogs`
   - `/admin/login`
   - CV download.

## Notes

- Keep Docker images and logs pruned because EC2 disk can fill quickly.
- Avoid running heavy builds during traffic spikes.
- Do not expose database ports publicly.
- Store all secrets in Coolify environment variables, not in Git.
