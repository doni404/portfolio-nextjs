# Coolify Server Setup Steps

## Goal

Prepare the production server for the portfolio app while frontend/backend integration is still in progress.

Target stack:

- AWS EC2.
- Amazon Linux 2023.
- Coolify.
- PostgreSQL managed by Coolify.
- GitHub repo: `https://github.com/doni404/portfolio-nextjs`.
- Frontend: Next.js.
- Backend: Express.js + Prisma.

## 1. Choose Instance Size

Recommended:

- Minimum for MVP: `t3.micro`.
- Better choice: `t3.small`.

Coolify official docs recommend at least 2 CPU cores, 2 GB RAM, and 30 GB storage. `t3.micro` has 2 vCPU but only 1 GB RAM, so add swap and monitor memory. If deployment or PostgreSQL feels unstable, upgrade to `t3.small`.

## 2. AWS Security Group

Open inbound:

- SSH `22` from your IP only.
- HTTP `80` from anywhere.
- HTTPS `443` from anywhere.
- Coolify setup port `8000` temporarily from your IP only.

Keep closed:

- PostgreSQL `5432`.
- App internal ports like `3000` and `4000`.

After Coolify is configured with HTTPS, remove or restrict direct `8000` access.

## 3. DNS Preparation

Recommended DNS records:

```text
doniputra.com          A      YOUR_EC2_PUBLIC_IP
www.doniputra.com      CNAME  doniputra.com
api.doniputra.com      A      YOUR_EC2_PUBLIC_IP
coolify.doniputra.com  A      YOUR_EC2_PUBLIC_IP
```

Simpler MVP option:

- Use `doniputra.com` for frontend.
- Use `api.doniputra.com` for backend.

This is easier in Coolify than same-domain `/api` routing.

## 4. Connect To Server

```bash
ssh ec2-user@YOUR_EC2_PUBLIC_IP
```

Update packages:

```bash
sudo dnf update -y
```

Install basic tools:

```bash
sudo dnf install -y curl git jq openssl
```

## 5. Add Swap

For `t3.micro`, add 2 GB swap:

```bash
sudo fallocate -l 2G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile
echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab
free -h
```

## 6. Install Coolify

Official quick install:

```bash
curl -fsSL https://cdn.coollabs.io/coolify/install.sh | sudo bash
```

After install, Coolify prints a URL such as:

```text
http://YOUR_EC2_PUBLIC_IP:8000
```

Open it immediately and create the first admin account.

Important: do this quickly. Before the first admin is created, anyone who can access the setup URL could claim the instance.

## 7. Initial Coolify Setup

In Coolify:

1. Create first admin user.
2. Confirm the local server is registered.
3. Configure server domain later, for example `coolify.doniputra.com`.
4. Connect GitHub account or GitHub app.
5. Select repository `doni404/portfolio-nextjs`.

## 8. Create Project

Create:

- Project: `portfolio-nextjs`
- Environment: `production`

Resources to create:

- PostgreSQL service.
- Backend app.
- Frontend app.

## 9. Create PostgreSQL In Coolify

Create a PostgreSQL resource:

```text
Name: portfolio-postgres
Database: portfolio
User: portfolio_app
Password: generated by Coolify
Public port: disabled
```

Copy the internal connection string for the backend `DATABASE_URL`.

## 10. Backend App Resource

Create an application from GitHub repo:

```text
Repository: doni404/portfolio-nextjs
Branch: main
Root directory: apps/api
Port: 4000
```

Suggested domains:

```text
api.doniputra.com
```

Environment variables:

```text
NODE_ENV=production
PORT=4000
DATABASE_URL=postgresql://portfolio_app:PASSWORD@portfolio-postgres:5432/portfolio
FRONTEND_URL=https://doniputra.com
CONTACT_NOTIFICATION_EMAIL=doniputrapurbawa@gmail.com
ADMIN_SEED_PASSWORD=replace-with-strong-initial-password
SESSION_SECRET=replace-with-long-random-secret
```

Build command depends on Cursor's final backend setup. Expected shape:

```bash
npm run prisma:generate && npm run build
```

Start command expected shape:

```bash
npm run start
```

Migration command should run before app start or as a pre-deploy command:

```bash
npm run prisma:migrate:deploy
```

Seed command should be manual, not automatic on every deploy:

```bash
npm run db:seed
```

## 11. Frontend App Resource

Create another application from the same GitHub repo:

```text
Repository: doni404/portfolio-nextjs
Branch: main
Root directory: apps/web
Port: 3000
```

Suggested domains:

```text
doniputra.com
www.doniputra.com
```

Environment variables:

```text
NODE_ENV=production
NEXT_PUBLIC_API_BASE_URL=https://api.doniputra.com
```

Build command:

```bash
npm run build
```

Start command:

```bash
npm run start
```

## 12. GitHub Actions

Use GitHub Actions for CI checks:

- lint
- typecheck
- unit tests
- API tests
- build

Let Coolify deploy from `main` after the code is merged.

Optional later:

- Disable Coolify auto-deploy.
- Add a GitHub Actions deploy job that calls a Coolify webhook only after CI passes.

## 13. SSL

In Coolify, enable HTTPS for:

- `doniputra.com`
- `www.doniputra.com`
- `api.doniputra.com`
- optionally `coolify.doniputra.com`

Coolify should manage the reverse proxy and Let's Encrypt certificates.

## 14. Backups

Enable PostgreSQL backups in Coolify.

Minimum:

- Daily backup.
- Keep 7 daily backups.
- Store backup outside the EC2 instance if possible.

Recommended:

- S3-compatible backup storage.
- Test restore once before launch.

## 15. Smoke Checks

After deployment:

```bash
curl -I https://doniputra.com
curl https://api.doniputra.com/api/health
```

Browser checks:

- Home page loads.
- `/blogs` loads.
- Blog detail loads.
- `/contact` loads.
- `/admin/login` loads.
- Contact form submits.
- Admin login works.
- CV download works.

## 16. Monitoring

Watch:

```bash
free -h
df -h
docker ps
docker stats
```

Upgrade to `t3.small` if:

- Builds fail.
- Coolify becomes slow.
- PostgreSQL restarts.
- Docker containers get killed.
- Memory is consistently high.

## 17. Do Not Do

- Do not expose PostgreSQL publicly.
- Do not commit `.env` files.
- Do not hardcode admin password.
- Do not auto-run seed on every production deploy unless it is fully safe and idempotent.
- Do not keep port `8000` open to the whole internet after setup.
